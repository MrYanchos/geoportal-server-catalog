A brief introduction to Data Discovery Studio interface

 

Here you enter a search term or phrase. DDH will search all indexed fields in all records (titles, abstracts, keywords, organizations, etc.) When you enter a phrase, such as coral bleaching, it will find documents where both coral and bleaching are found (it is equivalent to entering coral AND bleaching.) You can also search for documents that contain either coral or bleaching (type in coral OR bleaching), or use NOT to exclude terms. Notice that the search terms get italicized in the found metadata records.

Current filters will be shown here  - reflecting text search, spatial extent, temporal extent, and one or more filters reflecting selected facets.  Adding filters will narrows your search results. Uncheck the filters when you want to broaden the search.

Initially, the map filter is turned off – same as other facets. To enable spatial filter, select Intersects or Within, and zoom in on the area of interest. This will show only datasets with defined spatial extent that either intersects or is completely within the map area.

Here is a long list of additional options (facets) for narrowing your search results. You can zoom in on records by using time sliders (for Time Period, Date, Publication Date) and then clicking the temporal extent listed under the slider to add it as a filter, or extending search hierarchies that express various metadata dimensions. Some of these options (in bold) have been automatically constructed by the metadata enhancement pipeline behind the DDH system. The hierarchies used to categorize the metadata are based on several domain ontologies.

Search returns a list of metadata records. The records include title of the item described by the record (it could be a dataset, service, data portal, publication, etc.), beginning of an abstract, data source, the date it was ingested or modified in DDH, and a list of operations you can do over this record. These are described in items (6) – (12)

The first item (“HTML”) presents the metadata content in a human-readable form, including metadata elements automatically added by the CINERGI pipeline that underlies DDH. This information should help you decide if the dataset fits for your purpose.

Two additional views of the metadata (XML and JSON) present it in machine readable formats. Notice the URLs, you may need to use them in your code to access these data. Also, as you explore the XML view, search for “cinergi” to discover metadata enhancements generated by the system for this record. You can edit these enhancements using the Edit link.

Get Data lists URLs found in the metadata record. Depending on the type of data, they may point to a downloadable dataset, an online portal, a web service endpoint, respective documentation, etc. DDH tries to determine which of the links to point to as close to the data as possible, but in most cases it will err on the side of caution and include additional relevant links.

Data Discovery Studio is a new element of DDH. It allows you to pass the metadata record to a collection of notebooks served from a Jupyter Hub. Using Jupyter notebooks one can download and visualize the data, conduct analysis, and execute models. The collection of notebooks, as well as registered JupyterHub servers, is growing. Please explore it on your own.

If the metadata record has been enhanced using the CINERGI pipeline, this link will open a provenance view for the record. It will show graphically which metadata enhancers have been executed over the record, and what they produced.

Automatic metadata enhancement is error-prone. Using our Metadata Editor, you can edit metadata elements, accept or reject automatic metadata enhancement, and re-submit records for processing by the pipeline
